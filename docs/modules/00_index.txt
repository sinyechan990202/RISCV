================================================================================
  RV32I 32비트 명령어 인코딩 형식
================================================================================

  [ R-type ]  — 레지스터-레지스터 연산 (ADD, SUB, AND, OR, XOR, SLL, SRL, SRA, SLT)
  ┌─────────┬────┬────┬────────┬────┬────────┐
  │ [31:25] │[24:20]│[19:15]│[14:12]│[11:7]│[6:0]  │
  │ funct7  │ rs2   │  rs1  │funct3 │  rd  │opcode │
  │  7-bit  │ 5-bit │ 5-bit │ 3-bit │5-bit │ 7-bit │
  └─────────┴────────┴───────┴───────┴──────┴───────┘
  예) ADD x3, x1, x2  →  0000000_00010_00001_000_00011_0110011
                          funct7  rs2   rs1  f3   rd   opcode
      16진수: 0x0020_81B3

  [ I-type ]  — 즉치값 연산 / LOAD / JALR / CSR
  ┌────────────────┬───────┬────────┬───────┬────────┐
  │   [31:20]      │[19:15]│[14:12] │[11:7] │[6:0]   │
  │   imm[11:0]    │  rs1  │ funct3 │  rd   │ opcode │
  │    12-bit      │ 5-bit │ 3-bit  │ 5-bit │ 7-bit  │
  └────────────────┴───────┴────────┴───────┴────────┘
  예) ADDI x1, x0, 5  →  000000000101_00000_000_00001_0010011
                          imm=5        rs1=0 f3  rd=1  opcode
      16진수: 0x0050_0093

  예) LW x11, 0(x0)   →  000000000000_00000_010_01011_0000011
                          imm=0        rs1=0 f3  rd=11 opcode
      16진수: 0x0000_2583

  [ S-type ]  — STORE (SW, SH, SB)
  ┌────────┬───────┬───────┬────────┬────────┬────────┐
  │[31:25] │[24:20]│[19:15]│[14:12] │[11:7]  │[6:0]   │
  │imm[11:5]│  rs2  │  rs1  │ funct3 │imm[4:0]│ opcode │
  │ 7-bit  │ 5-bit │ 5-bit │ 3-bit  │ 5-bit  │ 7-bit  │
  └────────┴───────┴───────┴────────┴────────┴────────┘
  예) SW x3, 0(x0)    →  0000000_00011_00000_010_00000_0100011
                          imm[11:5] rs2  rs1  f3 imm[4:0] opcode
      16진수: 0x0030_2023

  [ B-type ]  — 조건 분기 (BEQ, BNE, BLT, BGE, BLTU, BGEU)
  ┌─────┬────┬───────┬───────┬────────┬────┬────────┐
  │[31] │[30:25]│[24:20]│[19:15]│[14:12]│[11:8]│[7]│[6:0]│
  │imm[12]│imm[10:5]│ rs2 │ rs1 │funct3 │imm[4:1]│imm[11]│opcode│
  └─────┴───────┴───────┴───────┴────────┴──────┴───┴──────┘
  예) BNE x1, x2, +8  →  0_000000_00010_00001_001_0100_0_1100011
      16진수: 0x0020_9463    (imm=+8, rs2=2, rs1=1, funct3=001)

  [ U-type ]  — 상위 즉치값 (LUI, AUIPC)
  ┌───────────────────────┬───────┬────────┐
  │       [31:12]         │[11:7] │[6:0]   │
  │      imm[31:12]       │  rd   │ opcode │
  │        20-bit         │ 5-bit │ 7-bit  │
  └───────────────────────┴───────┴────────┘
  예) LUI x10, 1  →  00000000000000000001_01010_0110111
                      imm[31:12]=1          rd=10 opcode
      16진수: 0x0000_1537   (x10 ← 0x00001000)

  [ J-type ]  — 무조건 점프 (JAL)
  ┌─────┬──────────┬────┬────────┬───────┬────────┐
  │[31] │[30:21]   │[20]│[19:12] │[11:7] │[6:0]   │
  │imm[20]│imm[10:1]│imm[11]│imm[19:12]│ rd  │ opcode │
  └─────┴──────────┴────┴────────┴───────┴────────┘
  예) JAL x0, 0 (halt)  →  0_0000000000_0_00000000_00000_1101111
      16진수: 0x0000_006F   (PC 자기 자신으로 점프 → 무한루프 halt)

  ─────────────────────────────────────────────────────────────────────────────
  주요 opcode 값 (하위 7비트)
  ─────────────────────────────────────────────────────────────────────────────
  0110011 = R-type (ADD/SUB/AND/OR/XOR/SLL/SRL/SRA/SLT/SLTU)
  0010011 = I-type ALU (ADDI/ANDI/ORI/XORI/SLLI/SRLI/SRAI/SLTI/SLTIU)
  0000011 = LOAD (LW/LH/LB/LHU/LBU)
  0100011 = STORE (SW/SH/SB)
  1100011 = BRANCH (BEQ/BNE/BLT/BGE/BLTU/BGEU)
  1101111 = JAL
  1100111 = JALR
  0110111 = LUI
  0010111 = AUIPC
  1110011 = SYSTEM (ECALL/EBREAK/CSR*)
  0001111 = FENCE (NOP으로 처리)

  ─────────────────────────────────────────────────────────────────────────────
  32비트 명령어 → 16진수 변환 예시
  ─────────────────────────────────────────────────────────────────────────────
  명령어                  | 2진수 (32비트)                         | 16진수
  ------------------------|----------------------------------------|-----------
  ADDI x1,  x0,  5       | 0000_0000_0101_0000_0000_0000_1001_0011| 0x00500093
  ADDI x2,  x0,  3       | 0000_0000_0011_0000_0000_0001_0001_0011| 0x00300113
  ADD  x3,  x1,  x2      | 0000_0000_0010_0000_1000_0001_1011_0011| 0x002081B3
  SUB  x4,  x3,  x2      | 0100_0000_0010_0001_1000_0010_0011_0011| 0x40218233
  AND  x5,  x3,  x4      | 0000_0000_0100_0001_1111_0010_1011_0011| 0x0041F2B3
  OR   x6,  x3,  x4      | 0000_0000_0100_0001_1110_0011_0011_0011| 0x0041E333
  SLL  x8,  x1,  x2      | 0000_0000_0010_0000_1001_0100_0011_0011| 0x00209433
  SRL  x9,  x8,  x2      | 0000_0000_0010_0100_0101_0100_1011_0011| 0x002454B3
  LUI  x10, 1            | 0000_0000_0000_0000_0001_0101_0011_0111| 0x00001537
  LW   x11, 0(x0)        | 0000_0000_0000_0000_0010_0101_1000_0011| 0x00002583
  SW   x3,  0(x0)        | 0000_0000_0011_0000_0010_0000_0010_0011| 0x00302023
  BNE  x1,  x2, +8       | 0000_0000_0010_0000_1001_0100_0110_0011| 0x00209463
  JAL  x0,  0 (halt)     | 0000_0000_0000_0000_0000_0000_0110_1111| 0x0000006F
  NOP (ADDI x0,x0,0)     | 0000_0000_0000_0000_0000_0000_0001_0011| 0x00000013
  AUIPC x18, 0           | 0000_0000_0000_0000_0000_1001_0001_0111| 0x00000917
  SRA x21, x20, x2       | 0100_0000_0010_1010_0101_1010_1011_0011| 0x402A5AB3
  ADDI x20, x0, -1       | 1111_1111_1111_0000_0000_1010_0001_0011| 0xFFF00A13

================================================================================
  파이프라인 데이터 흐름 요약
================================================================================

  [외부 imem]
       ↕ (instr_mem_if)
  IF Stage (if_stage)
       ↕ pc_reg
       ↓
  [IF/ID 레지스터]
       ↓
  ID Stage (id_stage)
    ├─ reg_file
    ├─ imm_gen
    └─ csr_regfile
       ↓
  [ID/EX 레지스터]
       ↓
  EX Stage (ex_stage)
    ├─ forwarding_unit ←── EX/MEM, MEM/WB 피드백
    ├─ alu
    └─ branch_unit
       ↓
  [EX/MEM 레지스터] ──→ forwarding_unit (EX→EX 소스)
       ↓
  MEM Stage (mem_stage)
       ↕ (data_mem_if)
  [외부 dmem]
       ↓
  [MEM/WB 레지스터] ──→ forwarding_unit (MEM→EX 소스)
       ↓
  WB Stage (wb_stage)
       ↓
  reg_file (쓰기 피드백) ──→ ID Stage

  hazard_unit ─── stall/flush/pc_sel → 모든 파이프 레지스터 및 PC

================================================================================
  해저드 처리 빠른 참조
================================================================================

  해저드 종류        | stall      | flush      | 포워딩
  ----------------|------------|------------|---------------------------
  Load-Use        | IF, ID     | EX         | 다음 사이클 MEM→EX
  CSR RAW         | IF, ID     | EX         | 없음 (reg_file 바이패스)
  Branch taken    | 없음        | ID, EX     | 없음
  JAL / JALR      | 없음        | ID, EX     | 없음
  EX→EX RAW       | 없음        | 없음        | forwarding_unit (즉시)
  MEM→EX RAW      | 없음        | 없음        | forwarding_unit (1-NOP 간격)
  imem not ready  | IF~WB 전체  | 없음        | 없음
  dmem not ready  | IF~WB 전체  | 없음        | 없음
  Trap            | 없음        | ID, EX     | 없음 (mtvec으로 PC 이동)
