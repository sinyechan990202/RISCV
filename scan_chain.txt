================================================================================
                         스캔 체인 (Scan Chain) 정리
================================================================================

1. 개요
--------------------------------------------------------------------------------
스캔 체인은 디지털 회로의 테스트 용이성 설계(DFT: Design for Testability) 기법이다.
칩 내부의 플립플롭(FF)들을 직렬로 연결(체인 구조)하여, 외부에서 내부 상태를
직접 읽고 쓸 수 있게 만드는 구조다.

양산 후 칩의 제조 결함(단선, 단락 등)을 검출하는 데 주로 사용된다.


2. 왜 필요한가
--------------------------------------------------------------------------------
일반적인 디지털 회로에서는 내부 FF의 상태를 직접 관측하거나 제어하기 어렵다.
예를 들어 10단 파이프라인의 3번째 FF 값을 외부에서 읽으려면
회로가 특정 입력을 받고 여러 클럭을 돌아야만 한다.

스캔 체인 없이 테스트하면:
  - 내부 노드를 직접 제어/관측 불가 → 테스트 커버리지 낮음
  - 결함 위치 파악 어려움 (진단 불가)
  - 가능한 입력 패턴이 기하급수적으로 증가 → 테스트 시간 폭증

스캔 체인이 있으면:
  - 모든 FF의 초기값을 임의로 설정 가능 (scan in)
  - 클럭 몇 번 후 모든 FF 값을 읽어낼 수 있음 (scan out)
  - ATPG 툴로 자동 패턴 생성 가능
  - 결함 위치 진단까지 가능


3. 핵심 구성 요소: 스캔 플립플롭 (Scan FF / MUX-D FF)
--------------------------------------------------------------------------------
일반 D-FF에 MUX를 하나 추가한 형태.

    [scan_in] ─┐
               ├── MUX ── D ── [FF] ── Q
    [data_in] ─┘     ↑
                   SE (Scan Enable)

  SE = 0 (Normal mode): data_in이 FF로 입력됨 → 회로가 정상 동작
  SE = 1 (Scan mode):   scan_in이 FF로 입력됨 → 이전 FF의 출력을 받아 체인 구성

이 FF들을 직렬로 연결하면 하나의 긴 시프트 레지스터가 된다.


4. 스캔 체인 동작 흐름
--------------------------------------------------------------------------------

  [Scan In 핀] → FF0 → FF1 → FF2 → ... → FFn → [Scan Out 핀]

  (1) Shift-in (스캔 입력 단계)
      - SE = 1
      - scan_in 핀으로 원하는 비트 패턴을 n 클럭 동안 shift
      - 각 FF에 테스트 초기값이 로드됨

  (2) Capture (캡처 단계)
      - SE = 0
      - 클럭 1~2 사이클 동안 정상 회로 동작 수행
      - 각 FF가 combinational logic의 출력값을 캡처

  (3) Shift-out (스캔 출력 단계)
      - SE = 1
      - scan_out 핀으로 n 클럭 동안 shift-out
      - 캡처된 FF 값들을 외부로 읽어냄
      - 기대값(expected)과 비교 → PASS / FAIL 판정

  위 과정을 여러 패턴으로 반복하면 내부 결함 검출 가능.


5. 스캔 체인과 ATPG
--------------------------------------------------------------------------------
ATPG (Automatic Test Pattern Generation):
  - 결함 모델(주로 Stuck-at-0/1)을 기준으로
  - 결함을 활성화(activate)하고 관측(propagate)할 수 있는 패턴을 자동 생성
  - 스캔 체인 구조가 있어야 내부 FF 제어/관측이 가능하므로 ATPG의 전제 조건

주요 결함 모델:
  - Stuck-at-0 / Stuck-at-1: 노드가 항상 0 또는 1로 고정
  - Transition fault:  느린 신호 전환 (타이밍 결함)
  - Bridging fault:    인접 신호선 단락


6. 멀티 체인 구조
--------------------------------------------------------------------------------
FF 수가 많으면 체인이 길어져 shift 시간이 증가한다.
이를 해결하기 위해 여러 개의 체인으로 병렬 분리한다.

  Scan In 0 → [chain 0] → Scan Out 0
  Scan In 1 → [chain 1] → Scan Out 1
  Scan In 2 → [chain 2] → Scan Out 2

  체인 수 k개로 나누면 shift 시간 ≈ 1/k로 단축.
  (핀 수는 2k개 증가)


7. 스캔 체인 관련 주요 신호
--------------------------------------------------------------------------------
  scan_in    : 스캔 데이터 입력 (체인의 시작)
  scan_out   : 스캔 데이터 출력 (체인의 끝)
  scan_en    : 스캔 모드 활성화 (SE 신호)
  scan_clk   : 스캔 전용 클럭 (또는 기능 클럭 공용 사용)
  test_mode  : 칩 전체를 테스트 모드로 전환하는 글로벌 신호


8. 파이프라인 RTL과의 관계
--------------------------------------------------------------------------------
5-stage 파이프라인(IF/ID/EX/MEM/WB)을 예로 들면:
  - 각 파이프 레지스터(IF/ID_reg, ID/EX_reg, ...)의 FF들이 스캔 체인에 포함됨
  - 스캔 모드에서 파이프 레지스터에 임의값을 주입 → 각 스테이지의
    combinational logic(ALU, 디코더 등)을 독립적으로 테스트 가능
  - 일반 기능 시뮬레이션(tb)은 스캔 체인을 사용하지 않으며,
    스캔 테스트는 제조 후 ATE(Automated Test Equipment)에서 수행됨


9. 요약
--------------------------------------------------------------------------------
  항목          | 내용
  --------------|----------------------------------------------------------
  목적          | 제조 결함 검출 (Manufacturing Defect Detection)
  핵심 구조     | MUX-D 스캔 FF를 직렬로 연결한 시프트 레지스터
  동작 3단계    | Shift-in → Capture → Shift-out
  연관 기술     | ATPG, DFT, BIST, Boundary Scan (JTAG IEEE 1149.1)
  사용 시점     | 칩 양산 후 웨이퍼/패키지 테스트 (기능 시뮬과 별개)
  커버리지 목표 | 일반적으로 99% 이상의 Stuck-at fault coverage
